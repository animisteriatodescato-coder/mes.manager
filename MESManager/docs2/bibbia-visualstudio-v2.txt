REGOLE ARCHITETTURALI INVIOLABILI
──────────────────────────────────
1. ZERO Duplicazione
   → Una sola fonte di verità

2. Coerenza Strutturale
   → Clean Architecture sempre

3. Ogni Modifica Indica:
   → File coinvolti
   → Impatti
   → Docs da aggiornare

4. Database
   → Dev ≠ Prod SEMPRE
   → Script SQL obbligatori
   → Migration per schema

5. Frontend
   → UX stabile (no regressioni)
   → Stato persistente

6. Deploy
   → MAI sovrascrivere appsettings.Secrets/Database.json
   → Versione SEMPRE incrementata (v1.23 → v1.24)
   → Ordine servizi: Stop(PlcSync→Worker→Web) Start(Web→Worker→PlcSync)

7. PLC
   → IP macchine in DATABASE
   → Offset in JSON
   → Graceful shutdown (libera slot)

8. Sicurezza
   → Secrets cifrati (DPAPI)
   → Ruoli rispettati
   → Parametrized queries
